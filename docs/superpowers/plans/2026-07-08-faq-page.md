# FAQ Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `/faq` page with collapsed FAQ sections and stable links to each section and question.

**Architecture:** Create a TanStack Router file route at `src/routes/faq.tsx`. Keep FAQ content in typed arrays so sample copy can be replaced without changing rendering logic. Use DaisyUI collapse components and hash handling similar to `src/routes/projects.tsx`.

**Tech Stack:** TanStack Router, React 19, DaisyUI/Tailwind CSS, TypeScript.

---

### Task 1: Add FAQ Route

**Files:**
- Create: `src/routes/faq.tsx`

- [ ] **Step 1: Define typed FAQ data**

Add `FaqSection` and `FaqItem` types, then define sample sections and items:

```tsx
type FaqItem = {
  id: string
  question: string
  answer: string
}

type FaqSection = {
  id: string
  title: string
  description: string
  items: FaqItem[]
}
```

- [ ] **Step 2: Create the route**

Use `createFileRoute('/faq')` with `seo()` metadata and `FaqPage` as the component.

- [ ] **Step 3: Implement hash-open behavior**

Each section and item gets an `id`. When the URL hash matches a section ID or item ID, open the matching collapse and scroll it into view.

- [ ] **Step 4: Render section and item links**

Render a section index at the top. Each section heading links to `#section-id`; each question exposes a link to `#item-id`.
